# 响应式设计文档

> 创建日期：2026-05-19

---

## 一、整体方案

本项目采用 **Desktop-first（桌面优先）** 策略，以桌面端为基准编写样式，再通过 `max-width` 媒体查询向下适配平板和移动端。

样式集中在 `frontend/src/styles/components.css` 中，共使用了 5 个断点：

| 断点 | 适用场景 |
|------|---------|
| 900px | 中等屏幕（admin 双栏编辑器） |
| 768px | 平板 |
| 640px | 手机 |
| 560px | 小屏手机（admin 统计卡片） |
| 480px | 超小屏（书架抽屉表单） |

---

## 二、各断点详情

### 2.1 max-width: 900px（中等屏幕）

主要影响管理后台页面：

```css
@media (max-width: 900px) {
  /* admin 仪表盘统计卡片：4 列 → 2 列 */
  .stats-grid {
    grid-template-columns: repeat(2, 1fr);
  }

  /* 文章编辑器：编辑/预览双栏 → 单栏 */
  .post-editor {
    grid-template-columns: 1fr;
  }
}
```

### 2.2 max-width: 768px（平板）

影响主要公开页面的布局结构：

```css
@media (max-width: 768px) {
  /* Hero 区：左右两栏 → 单列 */
  .hero-section {
    flex-direction: column;
  }

  /* 博客侧边栏隐藏，内容区占满 */
  .sidebar {
    display: none;
  }
  .blog-content {
    width: 100%;
  }

  /* Issue 看板：多列 → 单列 */
  .issue-board {
    grid-template-columns: 1fr;
  }

  /* 仪表盘统计图：多列 → 两列 */
  .chart-grid {
    grid-template-columns: repeat(2, 1fr);
  }
}
```

### 2.3 max-width: 640px（手机）

导航栏切换为汉堡菜单，部分网格进一步收窄：

```css
@media (max-width: 640px) {
  /* Navbar：隐藏桌面链接，显示汉堡菜单按钮 */
  .nav-links {
    display: none;
  }
  .nav-hamburger {
    display: flex;
  }

  /* 随想录时间轴：两列 → 单列 */
  .musing-timeline {
    grid-template-columns: 1fr;
  }

  /* 书架网格：最小列宽收窄，列数自动减少 */
  .book-grid {
    grid-template-columns: repeat(auto-fill, minmax(160px, 1fr));
  }
}
```

> 注：汉堡菜单的展开/收起交互逻辑在 JS 中实现，CSS 层仅负责显示/隐藏状态切换。

### 2.4 max-width: 560px（小屏手机）

针对 admin 统计卡片做进一步降级：

```css
@media (max-width: 560px) {
  /* admin 统计卡片：2 列 → 单列 */
  .stats-grid {
    grid-template-columns: 1fr;
  }
}
```

### 2.5 max-width: 480px（超小屏）

书架抽屉的表单布局降级：

```css
@media (max-width: 480px) {
  /* 书架抽屉表单：多列 → 单列 */
  .book-drawer .form-grid {
    grid-template-columns: 1fr;
  }
}
```

## 三、自适应网格（无断点）

部分网格组件使用 `repeat(auto-fill, minmax())` 实现无需断点的自动列数适配，列数随容器宽度自然伸缩：

```css
/* Roadmap 规划卡片 */
.roadmap-grid {
  grid-template-columns: repeat(auto-fill, minmax(220px, 1fr));
}

/* 技能展示网格 */
.skills-grid {
  grid-template-columns: repeat(auto-fill, minmax(240px, 1fr));
}

/* 博客文章卡片列表 */
.posts-grid {
  grid-template-columns: repeat(auto-fill, minmax(300px, 1fr));
}

/* 书架网格（默认状态） */
.book-grid {
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
}
```

## 四、流式字体（clamp）

使用 CSS `clamp()` 函数让字体大小随视口宽度平滑缩放，避免在断点处出现跳变：

```css
/* 首页主标题：视口越宽字越大，但限制在 3rem ~ 5.5rem 之间 */
.hero-name {
  font-size: clamp(3rem, 7vw, 5.5rem);
}

/* 文章详情页标题 */
.post-detail-title {
  font-size: clamp(1.6rem, 4vw, 2.5rem);
}
```

语法说明：`clamp(最小值, 首选值, 最大值)`

## 五、不足与可改进点

### 5.1 断点体系不统一

目前使用了 5 个分散的断点值（900 / 768 / 640 / 560 / 480px），建议统一提取为 CSS 变量或在 `theme.css` 中用注释约定：

```css
/* 建议在 theme.css 中统一定义（CSS 变量暂不支持媒体查询，可作为注释规范） */
/* --bp-md:  900px  — 中等屏幕 */
/* --bp-sm:  768px  — 平板      */
/* --bp-xs:  640px  — 手机      */
/* --bp-2xs: 480px  — 超小屏    */
```

### 5.2 部分断点仅覆盖 admin 页面

`560px` 断点目前只处理了 `.stats-grid` 的单列降级，覆盖范围较窄。若后续 admin 页面样式增多，可考虑将该断点合并到 `640px` 或用更通用的 class 名统一处理。

### 5.3 汉堡菜单耦合 JS 逻辑

移动端 Navbar 的展开/收起依赖 JavaScript 操控 class，CSS 仅做 `display` 切换。如需更好的无 JS 降级体验，可考虑借助 `<details>`/`<summary>` 或 CSS `:focus-within` 实现纯 CSS 方案。
