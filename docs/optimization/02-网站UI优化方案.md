# 网站 UI 优化方案

> 参考站点：`https://erp-ai-dlc.nocode.meituan.com/#/`
> 
> 目标：在保留本站深色 neon / terminal 个性气质的前提下，借鉴参考站点清晰的信息层级、模块化展示方式与产品化表达，提升整体专业感、一致性与可读性。

---

## 一、方案结论

本次 UI 优化不建议直接照搬参考站点的企业产品官网风格，而应采用以下方向：

- 保留当前网站的个人技术品牌辨识度
- 借鉴参考站点的模块化信息组织方式
- 将首页、内容页、专题页统一到一套设计语言中
- 让网站从“功能集合 + 内容陈列”升级为“个人技术系统站”

推荐的最终定位：

- 首页像产品门户
- 博客像知识库
- Roadmap / Practice / Issues 像系统面板
- About 页面保留 terminal 特色，但补足结构化表达
- 后台与前台共享同一套设计 token 与组件规范

---

## 二、参考站点可借鉴点

结合参考站点页面结构与展示方式，最值得借鉴的是以下几点：

### 1. 信息层级清晰
··
参考站点的典型结构为：

- 顶部导航
- 首屏价值主张
- 模块说明区
- 流程/机制区
- CTA 引导区

每一屏都遵循“标题 + 说明 + 卡片/步骤”的稳定节奏，用户阅读路径明确。

### 2. 模块化表达成熟

站点并非简单罗列功能，而是把内容包装为：

- 价值点
- 机制说明
- 流程步骤
- 入口卡片
- 关键动作按钮

这种表达方式适合迁移到本站的博客、Roadmap、Practice、Issue Bin 等模块。

### 3. 卡片信息密度高但不混乱

参考站点的卡片具备以下特点：

- 标题简短
- 说明一两行即可说明价值
- 层级分明
- 视觉风格统一

这非常适合作为本站“页面分组卡”“概览卡”“内容卡”的参考范式。

### 4. 页面节奏统一

参考站点不同分区虽然内容不同，但骨架基本一致，因此整体体验很整。

本站目前的问题不是缺内容，而是不同页面的展示方式各自为战，统一性还有提升空间。

## 三、当前网站的现状判断

### 1. 现有优势

- 深色 neon 风格辨识度强
- 内容板块丰富，页面类型完整
- 部分页面已具备“系统面板”气质，如 `Roadmap`、`Practice`、后台 `Dashboard`
- `About` 页的 terminal 风格具有记忆点

### 2. 当前主要问题

- 首页仍偏传统个人主页，缺少“站点总览”能力
- 页面标题层与说明层不统一，不同页面缺少同一套骨架
- 卡片体系较散，信息密度与样式规则没有完全统一
- 一些页面更像“功能页”，缺少“产品化表达”
- Footer 固定悬浮，容易影响底部阅读体验
- 某些页面虽然有内容，但没有形成“概览 -> 分组 -> 引导”的清晰阅读路径

### 3. 设计升级目标

- 从“个人站”提升到“个人技术系统站”
- 从“页面能用”提升到“结构明确、表达专业、风格统一”
- 从“视觉个性鲜明”提升到“视觉个性 + 信息组织都成熟”

---

## 四、整体设计策略

## 4.1 设计关键词

建议未来 UI 统一围绕以下关键词展开：

- 克制
- 稳定
- 科技感
- 结构化
- 可持续演进

## 4.2 视觉方向

建议采用：

- 大面积中性深色背景
- neon 色只做重点强调
- 卡片层次明显，但避免过度发光
- 动效减少噪声，保留关键反馈

## 4.3 结构方向

每个主要页面尽量统一为：

- 页面头部：标题 + 副标题 + 概览信息 / 操作区
- 页面主体：概览区 + 分组区 + 内容区
- 页面尾部：补充说明 / 下一步入口 / 相关推荐

---

## 五、视觉语言升级建议

### 5.1 色彩体系优化

当前主题色有明显 neon 特征，这一方向应保留，但需要降低主界面的高亮密度。

建议：

- 背景色以更稳的深色层次为主
- `cyan` 作为主强调色
- `purple` 作为次强调色
- `pink`、`green`、`orange` 更多用于状态和告警，不作为主界面主色

建议效果：

- 标题、按钮、边框不会同时抢注意力
- 页面整体更高级、更耐看
- 真正重点内容更容易被识别

### 5.2 卡片系统统一

建议将全站卡片统一收敛为三类：

#### A. 概览卡 `overview-card`

用于：

- 数据统计
- 页面摘要
- 当前状态
- 关键指标

适用页面：

- 首页
- Blog 列表
- Roadmap
- Practice
- Dashboard

#### B. 内容卡 `content-card`

用于：

- 博客卡片
- 书籍卡片
- 随想卡片
- 学习项卡片
- Issue 卡片

#### C. 面板卡 `panel-card`

用于：

- 页面说明
- 机制说明
- 流程步骤
- 筛选器容器
- 模块导航卡

### 5.3 卡片视觉规范

统一建议：

- 圆角：`16px ~ 20px`
- 边框：低对比边框，不要太亮
- 阴影：短阴影、轻悬浮
- hover：轻微上浮 + 边框强化 + 极弱辉光
- 标题与正文间距统一

### 5.4 动效原则

建议动效规则统一：

- 页面进入：轻微淡入与上移
- 卡片 hover：轻微抬升
- 按钮 hover：亮度变化 + 边框强化
- 背景装饰动效只在少数重点区域保留

避免：

- 大面积持续动画
- 多个区域同时发光抢焦点
- 每个组件各自使用不同节奏的 hover 反馈

## 六、首页优化方案

首页是这次升级中最值得优先改造的页面。

当前首页结构：

- HeroSection
- SkillsSection
- RecentPosts

建议重构为以下 5 段。

### 6.1 首屏 Hero 升级

目标：从“自我介绍”升级为“站点价值主张”。

建议结构：

- 左侧：
  - 个人定位主标题
  - 网站说明副标题
  - 主 CTA：阅读博客
  - 次 CTA：查看路线图 / 关于我
- 右侧：
  - 当前身份
  - 当前关注方向
  - 正在推进的主题
  - 站点模块入口概览

建议替换掉纯装饰型标签云主导的表达方式，让右侧更有“信息价值”。

### 6.2 新增“站点导航总览区”

建议新增一组卡片，把主要模块做成首页入口：

- 博客：技术文章与系统思考
- 修炼手册：学习路线与长期积累
- Roadmap：功能规划与进展公开
- Issue Bin：问题、想法与处理中事项
- 书页间 / 随想录：轻内容与长期记录

每张卡片建议包含：

- 模块名称
- 一句说明
- 关键数字
- 跳转按钮

### 6.3 技能区改版

当前技能区使用百分比进度条，表达相对模板化。

建议改为“能力维度卡片”：

- 后端工程
- 前端体验
- AI 辅助开发
- 内容表达
- 系统设计 / 工程化

每张卡片展示：

- 核心关键词
- 代表技术栈
- 当前实践方向

这样更真实、更专业，也更符合个人品牌表达。

### 6.4 最新内容区增强

建议“最新文章”升级为“最新更新”：

可混合展示：

- 最新博客
- 最近更新的 Roadmap
- 最近推进的 Practice
- 最近一条公开 musings

如果暂时不做混合流，也建议先强化博客卡片的信息密度。

### 6.5 新增“当前进行中”区块

建议增加一组状态卡，用来体现网站是持续演进中的：

- 当前在做的功能
- 当前学习中的主题
- 当前关注的问题
- 最近新增记录

这部分非常适合借鉴参考站点“流程化、状态化”的展示方法。

## 七、页面级优化建议

### 7.1 博客列表页 `BlogListPage`

建议目标：从“普通文章列表”升级为“内容库浏览页”。

优化建议：

- 顶部增加页面说明区
- 增加概览卡：文章数、分类数、标签数、最近更新时间
- 左侧筛选区收敛为统一的 `panel-card`
- 增加排序入口（最新 / 热门 / 最近更新）
- 文章列表卡片强化摘要、元信息与 hover 反馈

### 7.2 文章详情页 `BlogDetailPage`

建议目标：从“Markdown 详情页”升级为“正式阅读页”。

优化建议：

- 标题区独立成文章 Hero
- 展示分类、标题、摘要、发布时间、阅读量、标签
- 正文内容宽度适度收窄
- 条件允许时增加目录导航
- 底部增加相关文章 / 上一篇 / 下一篇 / 返回入口

### 7.3 Roadmap 页 `RoadmapPage`

建议目标：从“条目列表”升级为“公开规划面板”。

优化建议：

- 顶部新增 overview 区：完成率、已完成、计划中、高优先级数
- 以“分组卡片 + 条目卡片”构成更清晰层级
- 条目卡显示更明确的状态与优先级
- 在页面头部增加“规划原则”说明卡
- 强化页面整体的系统感和路线图感

### 7.4 Practice 页 `PracticePage`

建议目标：从“学习清单”升级为“学习操作系统”。

优化建议：

- 顶部增加 overview 区
- 分类分组更像知识地图而不是普通列表
- 卡片中补充学习目标、资源数量、阶段标签
- 外链资源展示为更清晰的按钮组
- 页面底部可加入“下一阶段重点方向”说明区

### 7.5 Issue Board 页 `IssueBoardPage`

建议目标：强化为真正的“状态看板”。

优化建议：

- 顶部增加概览统计
- 三列看板标题和数量更突出
- Issue 卡片层级更清楚，避免视觉拥挤
- 小屏场景改为纵向堆叠，避免固定高度问题

### 7.6 Bookshelf / Musings 页

建议统一升级为“专题页模板”：

- 页面头部：标题 + 副标题 + 统计信息
- 页面中部：筛选器 + 卡片分组
- 页面底部：相关推荐或站内跳转

重点不是增加复杂交互，而是增强信息组织的一致性。

### 7.7 About 页 `AboutPage`

建议保留 terminal 风格，但补充结构化信息模块。

推荐结构：

- 上半部分保留 terminal 动画与交互
- 下半部分增加：
  - 我是谁
  - 我在做什么
  - 我关注什么
  - 如何联系我

这样既能保留个性，也能兼顾首次访问者的理解效率。

## 八、导航与全局体验优化

### 8.1 Navbar 升级

当前导航可用，但还可以进一步增强产品感。

建议：

- 强化当前页面高亮态
- 增加次级入口（如 GitHub、Admin）
- 移动端改为抽屉式菜单
- 条件允许时加入模块简介型 hover / popover

### 8.2 Footer 改造

当前 Footer 固定悬浮，容易影响底部阅读体验。

建议：

- 改为正常文档流 Footer
- 内容结构化为：
  - 站点简介
  - 快速入口
  - 联系方式
  - 最后更新时间 / 版本信息

### 8.3 页面头部模板统一

建议全站为主要页面统一一套 page header：

- 标题
- 副标题
- 关键数字或状态摘要
- 可选操作区

这样会显著提升整体一致性。

### 8.4 状态反馈统一

建议统一：

- 空状态
- 错误状态
- 加载状态
- 无障碍属性

优先方向：

- 骨架屏优于单纯转圈
- 错误状态有重试入口
- Toast 与状态提示具备 aria 语义

## 九、设计系统层面的落地建议

建议在前端样式层面逐步沉淀出一套更完整的设计 token：

### 9.1 Token 建议

补充统一变量：

- 页面最大宽度
- 区块间距
- 卡片圆角级别
- 页面标题、副标题字号体系
- 边框透明度层级
- 阴影层级
- hover / active / focus 状态色

### 9.2 建议新增的样式原语

可逐步抽出以下基础样式类或组件规范：

- `page-header`
- `page-subtitle`
- `overview-grid`
- `overview-card`
- `panel-card`
- `content-card`
- `section-intro`
- `empty-state`
- `skeleton-block`

### 9.3 前后台统一策略

虽然前台和后台氛围不同，但建议共享：

- 间距系统
- 卡片系统
- 圆角与边框规则
- 排版节奏
- 状态色规范

让后台不只是“浅色版另一个系统”，而是“同一产品体系下的管理端”。

## 十、推荐实施优先级

为了控制改造成本，建议分三期推进。

### Phase 1：低成本高收益

优先级最高，建议先做：

- 首页信息层级重构
- 统一页面头部结构
- 统一卡片体系
- Footer 改为非固定布局
- Blog / Roadmap / Practice 页面增加 overview 区

### Phase 2：体验强化

在第一期完成后推进：

- 文章详情页阅读体验升级
- Navbar 升级
- 空状态 / 错误状态 / 骨架屏统一
- 动效收敛与 hover 规则统一
- 各页面补充副标题和说明文案

### Phase 3：品牌化完善

作为后续增强项：

- 首页加入更完整的“当前状态面板”
- 全站搜索或 command palette
- 页面插画/几何背景/弱动态装饰
- 前后台更完整的同源设计语言建设

---

## 十一、建议优先改造页面顺序

如果以产出比排序，建议依次改造：

1. 首页 `HomePage`
2. 博客列表页 `BlogListPage`
3. Roadmap 页 `RoadmapPage`
4. Practice 页 `PracticePage`
5. Footer / Navbar 全局组件
6. 文章详情页 `BlogDetailPage`
7. About 页 `AboutPage`
8. Issue / Bookshelf / Musings 等专题页

---

## 十二、验收标准

本轮 UI 优化完成后，建议至少达到以下效果：

- 首页能够在 10 秒内让首次访问者理解网站结构
- 任意主要页面都具备统一的标题、副标题、概览区与内容区节奏
- 卡片视觉语言统一，不同模块看起来属于同一系统
- 深色 neon 风格得到保留，但不再喧宾夺主
- Roadmap / Practice / Blog 能体现“系统站”而不只是单页功能
- Footer 不再遮挡内容，整体页面滚动体验更自然

---

## 十三、最终建议

推荐采用的核心方向是：

**保留个人风格，借鉴产品化表达，升级为“有系统感的个人技术品牌站”。**

不建议把本站改成纯企业官网，也不建议继续维持当前各页面风格各自演化的状态。

最合适的路线是：

- 视觉上延续现有辨识度
- 结构上学习参考站的层级与节奏
- 页面上建立统一的模块化表达
- 逐步沉淀为一套可持续演进的设计系统

- 首页 UI 改版任务拆解
- 全局样式 token 调整清单
