# 前端开发问题

## 1. 写新文章页面编辑区文字不可见 【未解决】

**现象**：`PostEditorPage` 的 MDEditor 编辑区左侧输入内容后看不到文字，右侧预览区正常显示。

**背景**：后台使用明亮主题（白色背景），MDEditor 的实际输入层 `.w-md-editor-text-input` 使用 `-webkit-text-fill-color: transparent` 隐藏原始文字，依赖 `.w-md-editor-text-pre` 的语法高亮层来显示内容。语法高亮颜色由 `--color-prettylights-*` 系列变量控制，需要 `data-color-mode="light"` 才能正确注入浅色模式的变量值。

**已尝试**：
- 外层 `div` 加 `data-color-mode="light"` → 无效
- 在 `MDEditor` 组件上直接加 `data-color-mode="light"` prop → 无效
- CSS 强制覆盖 `color` 和 `background` → 无效，因为文字本身是透明的

**待排查方向**：`@uiw/react-md-editor@4.1.0` 的 `data-color-mode` 注入机制是否有变化，或需要通过其他方式（如 `previewOptions`、主题 API）切换到 light 模式。

## 2. 仪表盘页面一直加载无内容

**现象**：进入仪表盘后页面持续转圈，没有内容出现。

**原因**：`DashboardPage` 使用 `adminGetStats().then(...)` 请求数据，但没有 `.catch()`。接口失败时 `stats` 状态永远是 `null`，页面一直停留在 loading 状态。

**解决**：加上错误处理，失败时显示提示而非永久 loading：
```tsx
useEffect(() => {
  adminGetStats()
    .then((r) => setStats(r.data))
    .catch(() => setError(true));
}, []);

if (error) return <div>加载失败，请刷新重试</div>;
if (!stats) return <LoadingSpinner fullPage />;
```
