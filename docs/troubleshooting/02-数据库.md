# 数据库问题

## 1. JDBC 连接失败：不支持 utf8mb4 字符编码

**错误信息**
```
java.sql.SQLException: Unsupported character encoding 'utf8mb4'
```

**原因**：`utf8mb4` 是 MySQL 的字符集名称，不是 Java 标准字符集名。JDBC URL 中 `characterEncoding` 参数传给的是 Java 的 `Charset`，必须使用 Java 能识别的名称（如 `UTF-8`）。

**解决**：修改 `application-dev.yml` 中的连接串：
```yaml
url: jdbc:mysql://localhost:3306/blog_db?useSSL=false&serverTimezone=Asia/Shanghai&characterEncoding=UTF-8&allowPublicKeyRetrieval=true
```

## 2. JDBC 连接失败：数据库密码错误

**错误信息**
```
java.sql.SQLException: Access denied for user 'root'@'localhost' (using password: YES)
```

**原因**：`application-dev.yml` 中密码字段填的是占位符 `your_password`，未替换为本机实际密码。

**解决**：修改 `backend/src/main/resources/application-dev.yml`：
```yaml
spring:
  datasource:
    password: 你的实际MySQL密码
```

验证本机 MySQL 密码是否正确：
```bash
mysql -u root -p
```

## 3. 数据库表字段缺失：Unknown column

**错误信息**
```
java.sql.SQLSyntaxErrorException: Unknown column 'type' in 'field list'
```

**原因**：`schema.sql` 中的 `CREATE TABLE IF NOT EXISTS` 只在表不存在时建表，不会为已存在的表补充新字段。后续新增字段的 `ALTER TABLE` 迁移语句需要手动单独执行一次。

**解决**：在 MySQL 中执行对应的迁移语句：
```sql
ALTER TABLE `post`
  ADD COLUMN `type`       VARCHAR(32) NOT NULL DEFAULT 'blog' COMMENT 'blog | ai_timeline' AFTER `cover_image`,
  ADD COLUMN `event_date` DATE        COMMENT 'AI 大事纪事件日期' AFTER `type`;

ALTER TABLE `post`
  ADD INDEX `idx_type_event_date` (`type`, `event_date` DESC);
```

**注意**：MySQL 9.x 不支持 `ADD COLUMN IF NOT EXISTS` 语法，去掉 `IF NOT EXISTS` 直接执行。若字段已存在会报错，跳过即可。

## 4. 登录失败：预设密码 hash 与实际密码不匹配

**现象**：使用预设账号 `admin / Admin@123456` 无法登录，但数据库中确实存在该用户记录。

**原因**：`schema.sql` 中硬编码的 BCrypt hash 在生成时使用的并不是 `Admin@123456`，导致校验失败。`INSERT IGNORE` 不会更新已存在的记录，所以修改 `schema.sql` 后已写入数据库的错误 hash 不会自动更正。

**解决**：直接在数据库执行 UPDATE 修正密码：
```sql
UPDATE `user`
SET `password` = '$2a$12$8lpqXbN1qxT1rHmLd1eJmuEE2JCmJcuLpnSoxBaJKD0ikOihJcMAi'
WHERE `username` = 'admin';
```
上述 hash 对应密码为 `Admin@123456`（BCrypt strength=12）。

**日后修改密码**：不要直接写 hash，用代码生成：
```java
System.out.println(new BCryptPasswordEncoder(12).encode("新密码"));
```
