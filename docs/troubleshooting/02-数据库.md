# 数据库问题

## 1. JDBC 连接失败：不支持 utf8mb4 字符编码

**错误信息**
```
java.sql.SQLException: Unsupported character encoding 'utf8mb4'
Caused by: com.mysql.cj.exceptions.WrongArgumentException: Unsupported character encoding 'utf8mb4'
```

**原因**：JDBC URL 中写了 `characterEncoding=utf8mb4`，这是 MySQL 的叫法，Java 不识别，应使用 Java 标准编码名。

**解决**：将 `application-dev.yml` 中的连接串改为：
```
characterEncoding=UTF-8
```

## 2. JDBC 连接失败：Access denied

**错误信息**
```
java.sql.SQLException: Access denied for user 'root'@'localhost' (using password: YES)
```

**原因**：`application-dev.yml` 中的数据库密码填的是占位符 `your_password`，未改为本机实际密码。

**解决**：修改 `backend/src/main/resources/application-dev.yml`：
```yaml
spring:
  datasource:
    password: 你的实际MySQL密码
```

验证密码是否正确：
```bash
mysql -u root -p
```

## 3. 数据库表字段缺失：Unknown column 'type'

**错误信息**
```
java.sql.SQLSyntaxErrorException: Unknown column 'type' in 'field list'
```

**原因**：`schema.sql` 末尾的迁移语句（`ALTER TABLE`）没有执行，数据库中 `post` 表缺少 `type` 和 `event_date` 字段。`schema.sql` 里的 `CREATE TABLE IF NOT EXISTS` 不会自动补充新字段，迁移语句需要手动执行一次。

**解决**：在 MySQL 中执行：
```sql
ALTER TABLE `post`
  ADD COLUMN `type`       VARCHAR(32) NOT NULL DEFAULT 'blog' COMMENT 'blog | ai_timeline' AFTER `cover_image`,
  ADD COLUMN `event_date` DATE        COMMENT 'AI 大事纪事件日期' AFTER `type`;

ALTER TABLE `post`
  ADD INDEX `idx_type_event_date` (`type`, `event_date` DESC);
```

**注意**：MySQL 9.x 不支持 `ADD COLUMN IF NOT EXISTS` 语法，`IF NOT EXISTS` 需去掉。

## 4. schema.sql 中的初始密码 hash 错误

**现象**：使用预设账号 `admin / Admin@123456` 无法登录，提示用户名或密码错误，但数据库中确实存在该用户。

**原因**：`schema.sql` 中硬编码的 BCrypt hash 与实际密码不匹配（hash 生成时使用的密码不是 `Admin@123456`）。

**解决**：重新生成正确的 hash 后更新数据库：
```sql
UPDATE `user`
SET `password` = '$2a$12$8lpqXbN1qxT1rHmLd1eJmuEE2JCmJcuLpnSoxBaJKD0ikOihJcMAi'
WHERE `username` = 'admin';
```
该 hash 对应密码为 `Admin@123456`。

**注意**：`INSERT IGNORE` 不会更新已存在的记录，修改 `schema.sql` 后已存在的数据需用 `UPDATE` 单独修正。
