# 网络与接口问题

## 1. 登录请求被 CORS 拦截

**错误信息**
```
Access to XMLHttpRequest at 'http://localhost:8080/api/auth/login' from origin
'http://localhost:5173' has been blocked by CORS policy:
No 'Access-Control-Allow-Origin' header is present on the requested resource.
```

**原因**：Spring Security 的过滤器链在 MVC 层之前执行，preflight `OPTIONS` 请求被 Security 拦截后直接返回 401，响应头中没有 `Access-Control-Allow-Origin`，导致浏览器报跨域错误。仅在 `WebMvcConfig` 中配置 CORS 不够，Security 层感知不到。

**解决**：
1. 在 `SecurityConfig` 中通过 `.cors()` 注册 `CorsConfigurationSource` Bean，让 Security 直接管理 CORS：
```java
.cors(cors -> cors.configurationSource(corsConfigurationSource()))
```

2. 显式放行所有 `OPTIONS` 请求：
```java
.requestMatchers(HttpMethod.OPTIONS, "/**").permitAll()
```

3. 删除 `WebMvcConfig` 中重复的 CORS 配置，避免冲突。

**注意**：`WebMvcConfig` 的 `addCorsMappings` 与 Security 的 `CorsConfigurationSource` 同时存在会相互干扰，二选一即可，推荐统一放在 Security 层管理。
